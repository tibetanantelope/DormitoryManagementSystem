# 学生宿舍管理系统环境和使用教程

## 1. 项目技术栈

本项目是前后端分离的学生宿舍管理系统。

- 后端：Spring Boot 2.6.3、MyBatis、MyBatis-Plus、MySQL
- 前端：Vue 3、Vue Router、Vuex、Element Plus、Axios、ECharts
- 数据库：MySQL
- 后端端口：`9090`
- 前端接口代理：前端请求统一走 `/api`，再代理到 `http://localhost:9090`

## 2. 目录说明

```text
DormitoryManagementSystem
├─ Dormitory_business/       后端 Spring Boot 项目
├─ vue/                      前端 Vue 项目
├─ dormitory.sql             数据库建表和初始化数据脚本
├─ 数据库.txt                数据库相关说明
└─ 环境和使用教程.md          本教程
```

## 3. 环境要求

建议安装以下环境：

- JDK 11 或以上
- Maven 3.6 或以上
- Node.js 14 或以上
- npm
- MySQL 5.7 或 8.x
- IntelliJ IDEA 或其他 Java IDE
- VS Code 或 WebStorm，可选

检查命令：

```bash
java -version
mvn -version
node -v
npm -v
mysql --version
```

## 4. 数据库配置

后端数据库配置文件：

```text
Dormitory_business/src/main/resources/application.properties
```

当前配置：

```properties
server.port=9090

spring.datasource.url=jdbc:mysql://123.60.133.142:3306/dormitory?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false
spring.datasource.username=test
spring.datasource.password=123456
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
```

如果使用本地数据库，可以改成类似：

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/dormitory?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false
spring.datasource.username=root
spring.datasource.password=你的数据库密码
```

首次部署时，创建数据库并导入 SQL：

```sql
CREATE DATABASE dormitory DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

然后导入项目根目录的：

```text
dormitory.sql
```

可以使用 Navicat、DataGrip、MySQL Workbench，或者命令行：

```bash
mysql -u root -p dormitory < dormitory.sql
```

## 5. 后端启动

进入后端目录：

```bash
cd Dormitory_business
```

方式一：使用 Maven 启动

```bash
mvn spring-boot:run
```

方式二：使用 IDEA 启动

1. 用 IDEA 打开 `Dormitory_business`。
2. 等待 Maven 依赖加载完成。
3. 找到启动类：

```text
Dormitory_business/src/main/java/com/example/springboot/SpringbootApplication.java
```

4. 运行 `SpringbootApplication`。

启动成功后，后端地址为：

```text
http://localhost:9090
```

## 6. 前端启动

进入前端目录：

```bash
cd vue
```

安装依赖：

```bash
npm install
```

启动开发服务器：

```bash
npm run serve
```

前端启动后，终端会显示访问地址，通常是：

```text
http://localhost:8080
```

如果端口被占用，Vue 会自动提示使用其他端口。

前端代理配置文件：

```text
vue/vue.config.js
```

当前配置：

```js
proxy: {
  '/api': {
    target: 'http://localhost:9090',
    changeOrigin: true,
    pathRewrite: {
      '^/api': ''
    }
  }
}
```

前端接口封装文件：

```text
vue/src/utils/request.js
```

其中 `baseURL` 是：

```js
baseURL: '/api'
```

所以前端写接口时使用 `/stu/find`、`/room/find` 即可，不要再手动加 `/api`。

## 9. 主要功能说明

### 管理员 admin

管理员可以进行系统级管理：

- 学生信息管理
- 宿舍管理员信息管理
- 楼宇信息管理
- 房间信息管理
- 调宿申请执行和完成
- 报修申请处理
- 公告管理

### 宿舍管理员 dormManager

宿舍管理员按管辖楼栋管理：

- 查看自己管辖楼栋的学生
- 管理自己管辖楼栋相关申请
- 查看楼宇和房间信息
- 不能修改楼宇信息
- 不能修改房间信息

例如 `dorm1` 只能查看和管理 1 号楼相关学生和申请。

### 学生 stu

学生可以：

- 查看个人信息
- 查看自己的宿舍
- 提交调宿申请
- 提交报修申请
- 查看自己的申请记录

## 10. 调宿申请流程

正常调宿流程：

1. 学生提交调宿申请，状态为 `未处理`。
2. 宿舍管理员审核申请。
3. 审核通过后，状态为 `通过`。
4. admin 点击 `执行调宿`。
5. 系统检查目标床位是否为空、性别是否匹配宿舍楼类型。
6. 校验通过后，释放原床位，占用目标床位，状态改为 `处理中`。
7. admin 点击 `完成`。
8. 申请状态改为 `已完成`。

状态含义：

```text
未处理：学生提交后等待宿管审核
通过：宿管审核通过，等待 admin 执行
驳回：宿管审核不通过
处理中：admin 已执行床位调整，等待确认完成
已完成：调宿流程结束
```

## 11. 宿舍性别约束

系统按楼宇备注区分宿舍类型：

```text
男宿舍
女宿舍
```

约束规则：

- 男宿舍楼只能入住男生。
- 女宿舍楼只能入住女生。
- 手动维护房间床位时会校验学生性别。
- admin 执行调宿时会校验目标宿舍楼类型和学生性别。
- 已入住学生修改性别时，不能改成和当前宿舍楼类型冲突的性别。

## 12. 常见问题

### 1. 前端请求接口失败

检查后端是否启动：

```text
http://localhost:9090
```

检查前端代理配置是否仍指向：

```text
http://localhost:9090
```

### 2. 数据库连接失败

检查：

- MySQL 服务是否启动
- 数据库名是否为 `dormitory`
- 用户名密码是否正确
- `application.properties` 中连接地址是否正确

### 3. 登录失败

确认登录身份是否选择正确：

- 管理员选择 `admin`
- 宿管选择 `dormManager`
- 学生选择 `stu`

账号和身份不匹配会导致登录失败。

### 4. 调宿状态写入失败

如果数据库中的 `adjust_room.state` 枚举没有包含新状态，会出现状态写入失败。

应确保字段包含：

```sql
ENUM('未处理','通过','驳回','处理中','已完成')
```

可执行：

```sql
ALTER TABLE adjust_room
MODIFY state ENUM('未处理','通过','驳回','处理中','已完成')
NOT NULL DEFAULT '未处理'
COMMENT '申请状态';
```

### 5. npm 无法运行

如果 PowerShell 报错：

```text
npm.ps1 cannot be loaded because running scripts is disabled
```

可以使用：

```bash
npm.cmd run serve
npm.cmd run build
```
